# Инструкция по созданию нового проекта .csproj

## Общие правила

1. При создании нового проекта добавь к нему свойства `RootNamespace` и `AssemblyName`
   Значения этих свойств должны быть одинаковы. У значения обязательно должен быть префикс `Manager.`, он нужен, чтобы
   работала авторегистрация зависимостей в DI
2. Текущая версия .net фреймворка - `net9.0`
3. Мы не используем неявные юзинги. Свойство `ImplicitUsings` должно иметь значение `disable`
4. Используем `nullable enable` для прозрачности и избегания непредвиденных ошибок. `Nullable`свойство имеет значение
   `enable`
5. Стараемся придерживаться чистого билда без предупреждений, поэтому свойство `TreatWarningsAsErrors` ставим в `true`
6. Сомневаешься - подгляди в соседний проект такого же типа
7. Чтобы избежать конфликта версий используемых пакетов, используй ссылочную зависимость на Core проекты с нужными
   пакетами

## Тестовый проект

1. Проставляем свойство `IsTestProject` в `true`
2. Делаем ссылочную зависимость на инфраструктурные проекты тестов соответствующего типа
   1. Для юнит тестов это `Manager.Core.UnitTestsCore`
   2. Для интеграционных тестов это `Manager.Core.IntegrationTestsCore.csproj`
