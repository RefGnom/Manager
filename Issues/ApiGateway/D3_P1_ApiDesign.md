# Проектирование апи для сервиса ApiGateway

## Описание

ApiGateway обобщает в себе все возможности наших микросервисов и предоставляет единую точку входа.
Соответственно, нужно просмотреть все наши микросервисы и их апи перенести в ApiGateway

> Возможно, некоторые из микросервисов предоставляют апи не по стандарту REST, апи входной точки же должно
> соответствовать стандарту REST. Поэтому, при необходимости, нужно не 1 в 1 переносить апи методы из микросервисов, а
> подгонять под стандарт.

## Шаги

* [ ] Создать файл Documentation/ApiGateway/API.md
* [ ] В созданном файле описать все текущие возможности системы (все апи методы)
* [ ] Привести все методы к стандарту REST
* [ ] Отправить команде на ревью

## Ожидаемый результат

Должно быть спроектировано API для сервиса ApiGateway, которое любой разработчик сможет перенести в код сервиса
